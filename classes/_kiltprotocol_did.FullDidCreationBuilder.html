      The KeystoreSigner to sign the DID operation. It must contain the expected DID authentication key.

Static fromLightDidDetails

  • fromLightDidDetails(api: ApiPromise, details: LightDidDetails, upgradeOptions?: { withEncryptionKey: undefined | boolean; withServiceEndpoints: undefined | boolean }): FullDidCreationBuilder
    • Defined in did/src/DidBatcher/FullDidCreationBuilder.ts:82

    Initialize a DID creation with the information contained in the provided light DID.

    All the details in the DID are marked as to-add in the resulting creation operation.

    Parameters

    • api: ApiPromise

      The ApiPromise object to encode/decoded types as needed.

    • details: LightDidDetails

      The LightDidDetails object.

    • upgradeOptions: { withEncryptionKey: undefined | boolean; withServiceEndpoints: undefined | boolean } = {}

      Optional.

      • withEncryptionKey: undefined | boolean

        When set to true (default) the LightDID's encryption key is added to the on-chain DID.

      • withServiceEndpoints: undefined | boolean

        When set to true the LightDID's ServiceEndpoints are added to the on-chain DID. This is strictly opt-in as there are more restrictive size limits for on-chain service records.

    Returns FullDidCreationBuilder

    The builder initialized with the information contained in the light DID.
