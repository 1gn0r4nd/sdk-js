Class DidBatchBuilder

A builder to batch multiple DID-authorised extrinsics which are to be submitted by the same KILT account.

addSingleExtrinsic

  • addSingleExtrinsic(extrinsic: Extrinsic): DidBatchBuilder
    • Defined in did/src/DidBatcher/DidBatchBuilder.ts:105

    Add a new extrinsic to the batch.

    If the new extrinsic requires a different key type than the previous one, a new batch is created internally. Order of all extrinsics is maintained throughout the lifetime of the builder and when they are submitted to the KILT blockchain.

    The requirements to add a new extrinsic to the batch are the following:

    • The extrinsic must not be a DID management extrinsic. For those, FullDidCreationBuilder and FullDidUpdateBuilder must be used
    • The extrinsic must require a DID origin. E.g., staking extrinsics that require a simple KILT account origin cannot be added to the batch
    • The DID must have at least one key candidate to sign the provided extrinsic. If the DID is updated after the extrinsic is added but before the final batch is created, it results in undefined behavior, and most likely the extrinsic submission will fail.

    Parameters

    • extrinsic: Extrinsic

      The Extrinsic to add to the batch.

    Returns DidBatchBuilder

    The builder containing the new extrinsic in the last position of the internal queue.

build

  • build(signer: KeystoreSigner<any>, submitter: string, submissionOptions?: { atomic?: boolean; initialNonce?: BN; keySelection?: BatchSigningKeySelection }): Promise<SubmittableExtrinsic>
    • Defined in did/src/DidBatcher/DidBatchBuilder.ts:144

    Generate the SubmittableExtrinsic containing the batch of extrinsics to execute, in the order they were added to the builder.

    Parameters

    • signer: KeystoreSigner<any>

      The KeystoreSigner to sign the DID operation. It must contain the required keys to sign each batch.

    • submitter: string

      The KILT address of the user authorised to submit each extrinsic in the batch.

    • submissionOptions: { atomic?: boolean; initialNonce?: BN; keySelection?: BatchSigningKeySelection } = {}

      The additional options to customise the signing operation.

      • Optional atomic?: boolean

        A flag indicating whether the whole batch must be reverted (true) or not (false) in case any extrinsic in the batch fails. It defaults to true.

      • Optional initialNonce?: BN

        The initial nonce to use for the first batch, after which the nonce is increased by the builder. It defaults to the next valid DID nonce as stored on chain at the time this function is called.

      • Optional keySelection?: BatchSigningKeySelection

        The BatchSigningKeySelection callback to specify the DID key to use for each batch. It defaults to [[defaultBatchSigningKeySelectionCallback]].

    Returns Promise<SubmittableExtrinsic>

    The SubmittableExtrinsic containing the batch of batches.
