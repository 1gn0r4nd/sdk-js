Class RequestForAttestation

Requests for attestation are a core building block of the KILT SDK. A RequestForAttestation represents a Claim which needs to be validated. In practice, the RequestForAttestation is sent from a claimer to an attester.

A RequestForAttestation object contains the Claim and its hash, and legitimations/delegationId of the attester. It's signed by the claimer, to make it tamper-proof (claimerSignature is a property of Claim). A RequestForAttestation also supports hiding of claim data during a credential presentation.

Static verifySignature

  • verifySignature(input: IRequestForAttestation, verificationOpts?: { challenge?: string; resolver?: IDidResolver }): Promise<boolean>
    • Defined in core/src/requestforattestation/RequestForAttestation.ts:222

    [STATIC] [ASYNC] Verifies the signature of the RequestForAttestation object. It supports migrated DIDs, meaning that if the original claim within the RequestForAttestation included a light DID that was afterwards upgraded, the signature over the presentation must be generated with the full DID in order for the verification to be successful. On the other hand, a light DID that has been migrated and then deleted from the chain will not be allowed to generate valid presentations anymore.

    throws

    ERROR_IDENTITY_MISMATCH if the DidDetails do not match the claim owner or if the light DID is used after it has been upgraded.

    Parameters

    • input: IRequestForAttestation

      The RequestForAttestation.

    • verificationOpts: { challenge?: string; resolver?: IDidResolver } = {}

      Additional options to retrieve the details from the identifiers inside the request for attestation.

      • Optional challenge?: string

        The expected value of the challenge. Verification will fail in case of a mismatch.

      • Optional resolver?: IDidResolver

        The resolver used to resolve the claimer's identity. Defaults to DidResolver.

    Returns Promise<boolean>

    Whether the signature is correct.
