getAccountSignedAssociationExtrinsic

  • getAccountSignedAssociationExtrinsic(account: string, signatureValidUntilBlock: AnyNumber, signature: `0x${string}` | Uint8Array, sigType: "Ed25519" | "Sr25519" | "Ecdsa"): Promise<Extrinsic>
    • Defined in did/src/DidLinks/AccountLinks.chain.ts:188

    Signing (authorizing) this extrinsic with a FullDid and submitting it with any Account will link Account to FullDid and remove any pre-existing links of Account. Account must give permission by signing a Scale-encoded tuple consisting of the FullDid identifier and a block number representing the expiration block of the signature (after which it cannot be submitted anymore). Account does not need to hold balance. The submitting account will pay and own the deposit for the link.

    Parameters

    • account: string

      The account to link to the authorizing FullDid.

    • signatureValidUntilBlock: AnyNumber

      The link request will be rejected if submitted later than this block number.

    • signature: `0x${string}` | Uint8Array

      Account's signature over (DidIdentifier, BlockNumber).toU8a().

    • sigType: "Ed25519" | "Sr25519" | "Ecdsa"

      The type of key/substrate account which produced the signature.

    Returns Promise<Extrinsic>

    An extrinsic that must be did-authorized.

getAuthorizeLinkWithAccountExtrinsic

  • getAuthorizeLinkWithAccountExtrinsic(accountAddress: string, didIdentifier: string, signingCallback: LinkingSignerCallback, nBlocksValid?: number): Promise<Extrinsic>
    • Defined in did/src/DidLinks/AccountLinks.chain.ts:280

    Builds an extrinsic to link account to a did where the fees and deposit are covered by some third account. This extrinsic must be authorized using the FullDid whose didIdentifier was used here. Note that in addition to the signing account and did used here, the submitting account will also be able to dissolve the link via reclaiming its deposit!

    Parameters

    • accountAddress: string

      Address of the account to be linked.

    • didIdentifier: string

      Method-specific identifier FullDid to be linked.

    • signingCallback: LinkingSignerCallback

      The signature generation callback that generates the account signature over the encoded (DidIdentifier, BlockNumber) tuple.

    • nBlocksValid: number = 10

      How many blocks into the future should the account-signed proof be considered valid?

    Returns Promise<Extrinsic>

    An Extrinsic that must be did-authorized by the FullDid whose identifier was used.
